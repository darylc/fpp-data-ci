What happens after I submit?
  1. The issue is opened under your GitHub account, so you're notified automatically on any reply — no one needs to @-mention you.
  2. Two automated checks run within a minute or two and comment directly on the issue: one confirms your contact token is well-formed, the other clones your repo and runs the same compliance linter used for already-listed plugins — but a little stricter, since a new plugin doesn't get the grandfathering an existing one has. If it finds anything blocking, the comment tells you exactly what and where.
  3. If the scan asks for changes, push a fix to your repo, then comment /recheck on the issue to re-run it — no need to edit the issue itself.
  4. Once the checks pass, a maintainer still reviews it by hand — category fit, description wording, and anything the automation can't judge — before opening a pull request that adds your plugin to pluginList.json.
  5. Once that PR merges, your plugin appears in everyone's Plugin Manager the next time FPP refreshes its list.
