Request Plugin Removal

Requests should be submitted by the plugin owner.

Use this if you no longer want your plugin listed in FPP's Plugin Manager — existing installs are unaffected, it just stops being offered for new installs.

What happens after I submit?

  1. On the next page, tell us whether you're the plugin's author/maintainer and why you're requesting removal, then submit the issue.
  2. If you're the author, ownership is checked automatically (it compares your GitHub login against the plugin repo's owner). If it can't confirm that directly (e.g. an org-owned repo), it'll ask you to add "delist": true to your pluginInfo.json as proof — only someone with write access could do that.
  3. If you're a third party reporting an abandoned plugin, it's never removed automatically — the actual owner gets a chance to respond first, and a maintainer reviews it if there's no resolution.
  4. Once ownership is confirmed, the entry is removed from pluginList.json and the issue is closed automatically — no maintainer wait needed for a verified request.
